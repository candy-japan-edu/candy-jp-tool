# Candy 校内考库数据治理规则

校内考日期、出愿要求和考试形式每年都可能变化。这个 H5 可以先用样例数据跑通产品，但正式上线数据必须按字段风险分层维护。

## 字段分级

### A 级：学校级稳定字段

可批量维护，但仍需来源：

- 学校中文名 / 日文名
- 国公立 / 私立
- 地区
- 官网 URL
- 学部名 / 专业名

推荐来源：学校官网、学部页面、招生页面。

### B 级：入试要项高风险字段

必须逐年度核验：

- 出愿开始 / 出愿截止
- 校内考日期
- 合格发表日
- 出愿材料
- EJU 科目
- 英语成绩要求
- 其他硬性要求

推荐来源：当年度私费外国人留学生募集要项、学部入试页面、官方 PDF。

### C 级：考试内容字段

不得从往年或别的学部直接推断：

- 考试形式
- 校内考科目
- 面试 / 小论文 / 实技 / 作品集要求
- 二段选拔说明

### D 级：Candy 判断字段

必须标注为顾问判断，不能写成官方结论：

- Candy 点评
- 推荐度
- 竞争强弱
- 华人友好度
- 适合人群
- 备考建议

## 录入流程

1. 从 `data/schools-template.csv` 建表，先填学校、学部、专业、官网。
2. 每条专业只引用当年度官方募集要项，不用论坛二手信息当最终依据。
3. 日期字段填 `YYYY-MM-DD`；暂未核验就留空，不填猜测日期。
4. 数组字段在 CSV 中用英文逗号分隔。
5. 更新 JSON 前保留原始表格和核验记录。
6. 每次上线前抽查首页时间轴、详情页时间卡、对比表差异高亮。

## 页面表达规则

- 未核验数据必须显示为样例或待确认。
- 具体分数用“参考”或“建议”，不要写成保证录取。
- Candy 点评可以给策略判断，但不能替代官方要项。
- 医学、齿学等高风险专业必须额外提示按最新官网确认。
