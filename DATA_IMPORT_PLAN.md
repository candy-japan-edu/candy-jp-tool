# Candy 校内考库数据导入方案

目标是把腾讯文档中的学校专业表，稳定转换成 H5 可读取的 `data/schools.json`，并能持续迭代。

## 推荐表格

直接使用：

- `data/schools-template.csv`：空模板
- `data/schools-sample.csv`：首批 30 条样例
- `data/schools.schema.md`：字段解释

## 导入流程

1. **表格录入**
   在腾讯文档中按模板建表，所有日期使用 `YYYY-MM-DD`。

2. **字段核验**
   每条专业至少核验出愿截止、校内考日期、合格发表日、EJU 科目、英语要求。

3. **导出 CSV**
   从腾讯文档导出 CSV，保持表头不变。

4. **转换 JSON**
   将 CSV 转为 `data/schools.json`。数组字段需要转成 JSON 数组，布尔字段需要转成 `true` / `false`。

5. **本地预览**
   启动静态服务器，检查首页、详情、对比页。

6. **推送部署**
   提交到 GitHub 仓库，Vercel 自动部署。

## 数据批次建议

### 第一批

第一档 10 所 × 3 专业，共 30 条，作为获客工具首版。

### 第二批

补 MARCH、关关同立、地方旗舰国公立、常见保底私立，优先覆盖家长高频咨询学校。

### 第三批

补艺术、医学、齿学、药学、护理等需要特殊考试形式的方向。

## 工程建议

- Demo 阶段保持纯静态 JSON。
- 数据量超过 300 条后，可增加 `data/index.json` 做首页轻量索引。
- 真实上线后建议增加 `source_url`、`source_year`、`verified_by`、`verified_at` 字段。
- 如果接飞书多维表格 webhook，只负责咨询预约提交，不直接改核心招生数据。
